# HANDOFF — T-RO 2026 paper (FCP-MPC)

Portable context so a fresh Claude Code session on **another machine sharing this
git repo** can continue. (Claude Code conversation history + `~/.claude` memory do
**not** sync via git; this file does.)

## How to continue on the other machine
1. `git pull` (get code/tex/scripts/figures).
2. `conda activate cp` — **all sims need this env** (numpy/pybullet/skimage/sklearn).
   `python3`/system python lack numpy. `fitz` (PDF render) is only on
   `/opt/homebrew/bin/python3` here; on the other machine use whatever has PyMuPDF.
3. Start Claude Code in the repo and say: "read HANDOFF.md and continue."
4. Heavy experiments: `bash run_3d_experiments.sh` (see it for per-step commands).
   LaTeX build: `cd T_RO2026 && latexmk -pdf main.tex`.

## Key decisions already made
- **2D FCP**: online ACP envelope adaptation works (observations wired in
  `sims/sim_func_cp.py`); efficient via anchor-point projection in
  `controllers/func_cp_mpc.py`. Soft variant uses `w_safety=100` (tuned, in
  `runner_2d.py`).
- **3D FCP = offline hard ONLY** (no online adaptation): `FunctionalCPMPC3D` gated
  by `adaptive=False` (default) in `controllers/func_3d_mpc.py`. Rationale: ample
  3D free space; offline-hard suffices and is far faster (~50 ms vs ~1000 ms/step).
- **3D ACP-MPC** newly ported: `controllers/acp_3d_mpc.py` + `sim_acp_3d.py`,
  registered as method `acp` in `runner_3d.py`. Goal-biased holonomic sampler +
  ACI scalar radius. It legitimately times out in dense 3D (a feature: motivates
  the fieldwise FCP advantage that grows with dimension).
- Tables/legends ordered by dev year: **ACP, CC, ECP, FCP**.
- Known theory↔code note: ICS code uses `r_safe - Δ_i` (tracking margin **ε=0**);
  paper writes `r_safe + ε - Δ_i`. Decide: add ε in code or state ε=0 in paper.
